# 贡献指南

感谢你考虑为 AI Test Case Generator 做出贡献！

## 🤝 如何贡献

我们欢迎各种形式的贡献，包括但不限于：

- 🐛 报告 Bug
- 💡 提出新功能建议
- 📝 改进文档
- 🔧 提交代码修复
- 🌍 帮助翻译

## 🚀 开发环境设置

1. Fork [AItest-case-generator](https://github.com/yourname/AItest-case-generator) 仓库
2. 创建你的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交你的改动 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 创建一个 Pull Request

## 📋 代码规范

### Python 代码
- 遵循 PEP 8 规范
- 使用有意义的变量名和函数名
- 添加必要的注释和文档字符串
- 保持函数简洁，每个函数只做一件事

### 提交信息格式
```
<type>(<scope>): <subject>

<body>

<footer>
```

类型（type）：
- `feat`: 新功能
- `fix`: 修复 bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建/工具相关

示例：
```
feat(web-scraper): 支持动态网页渲染

- 添加 Playwright 集成
- 支持等待时间配置
- 修复内存泄漏问题

Closes #123
```

## 🐛 报告 Bug

在提交 Issue 之前，请确保：

1. 搜索已有的 Issue，避免重复
2. 使用 Bug Report 模板
3. 提供详细的复现步骤
4. 包含错误日志和系统信息

## 💡 功能建议

提交新功能建议时，请：

1. 清晰描述功能用途
2. 说明使用场景
3. 如果可能，提供设计思路或伪代码
4. 考虑功能的影响范围

## 📖 文档改进

文档改进包括：

- 修正错别字和语法错误
- 添加使用示例
- 改进说明的清晰度
- 补充缺失的文档部分

## ⚠️ 行为准则

- 尊重所有贡献者
- 保持友好和专业的交流
- 避免使用冒犯性语言
- 关注问题本身，而非个人
